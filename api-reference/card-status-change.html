Card Status Change​

This custodial integrator webhook endpoint is called when a card status is changed.

For more information on setting up webhooks see the webhooks guide

  • application/json
Request Body required
  • cardId string required

    The card ID

  • accountId string required
  • cardReference string required

    The card reference on the integrator side

  • status string required

    Possible values: [active, inactive, shipped, blocked, canceled]

  • reasonCode string required

    The reason code for the card status change

  • reasonMessage string required

    The reason message for the card status change

  • previousStatus string required

    Possible values: [active, inactive, shipped, blocked, canceled]

    The previous card status

  • updateDate date-time required

    The date and time of the card status change

Responses
  • 200

Card status change request accepted
